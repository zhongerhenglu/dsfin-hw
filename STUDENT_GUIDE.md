# 作业提交指南（简版）

> 详细说明请参阅 [STUDENT_GUIDE_Full.md](./STUDENT_GUIDE_Full.md)

---

## 你只需要做三件事

### 第一步：Fork 本仓库

打开本仓库页面，点击右上角 **Fork → Create fork**，将仓库复制到你的账号下。

> 每组只需一名同学操作，使用固定账号，**中途不要换账号**。

---

### 第二步：Clone 到本地

1. 打开 **GitHub Desktop**
2. 菜单 **File → Clone repository**
3. 在 **GitHub.com** 标签页找到你 Fork 的 `dsfin-hw`，点击 **Clone**

---

### 第三步：放入作业，提交，发起 PR

**放入作业：**

用文件管理器在本地仓库中创建你们小组的文件夹，路径格式：

```
submissions/2026/T01-G03-资产定价/
```

将作业文件放入该文件夹，**文件夹内必须包含一个 `README.md`**（说明小组成员、分工和作业摘要，格式参见 [STUDENT_GUIDE.md](./STUDENT_GUIDE.md)）。

**提交并推送：**

1. 打开 GitHub Desktop，左侧会显示所有变更文件
2. 左下角填写 Summary，例如：`T01-G03 第一次提交`
3. 点击 **Commit to main**，再点击 **Push origin**

**发起 Pull Request：**

1. GitHub Desktop 顶部点击 **Create Pull Request**，浏览器会自动打开
2. 将 PR 标题改为：`[2026/T01-G03] 资产定价 · 第一次提交`
3. 点击 **Create pull request**，等待教师审核合并

---

## 后续提交

首次 PR 合并后，后续更新**不需要再发起 PR**：

1. 修改或新增文件
2. GitHub Desktop 中 Commit → **Push origin**
3. 完成，系统自动同步

---

## 注意事项

- 文件夹命名格式：`T{话题编号}-G{小组编号}-{简短主题}`，例如 `T01-G03-资产定价`
- **不要修改其他小组的文件夹** (如果想要给其他小组提意见，可以在 GitHub 上直接评论)
- 单个文件请勿超过 50MB，大文件处理方式见 [STUDENT_GUIDE.md § 三](./STUDENT_GUIDE_full.md)
- 付费数据库（Wind、CSMAR 等）的数据**禁止直接上传** (可以通过坚果云等方式分享数据文件，并在 `README.md` 中说明数据来源和处理方法；坚果云的链接可以在课程群中发给任课老师)
